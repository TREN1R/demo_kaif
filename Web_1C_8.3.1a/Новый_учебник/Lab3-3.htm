Требования к  алгоритму

1)  Поля Склад, Контрагент и Валюта заполняются предопределенными реквизитами при открытии формы. При выборе договора должен открываться список только тех договоров, какие привязаны к выбранному Контрагенту. Фамилия материально ответственного лица должна выводиться при открытии формы. Заголовочная часть таблицы ТОВАРЫ должна содержать заголовки Цена и Сумма , СтавкаНДС и СуммаНДС в одном столбце друг под другом. Таблицы Товары и Услуги должна иметь подвальную часть для вывода итогов по колонке ВСЕГО.

2)  В табличной части Товары формы документа должна рассчитываться сумма налога. При подсчете колонки ВСЕГО сумма  должна суммироваться с размером налога.

3)  Цена товара  должна выбираться из регистра сведений Цены, но при необходимости может изменяться вручную, так же как и ставка НДС. Реквизит Количество вводится вручную. Любое изменение в строке табличной части должно приводить к пересчету колонки Всего.

4)  При формировании строк таблицы Услуга размер ставки НДС  всегда равен 18%. Цена услуги вводится вручную. Колонка всего рассчитывается путем сложения суммы налога и стоимости  услуги. Таблица имеет подвальную часть для вывода итогов.

5) Форма списка документа должна содержать поле СуммаДокумента, куда будет размещаться итоговая сумма по таблицам Товары и Услуги. Поле СуммаДокумента в форме документа должно отсутствовать.

Описание процедуры №1 (При изменении номенклатуры)

Алгоритм при изменении номенклатуры состоит из двух процедур и двух функций. Процедура ТоварыНоменклатураПриИзменении() становится активной при выборке из справочника Номенклатура нужной строки. В переменную СтрокаТаблицы заносится ссылка на активную строку таблицы, затем ведется обращение к функции ТоварыНоменклатураПриИзмененииНаСервере(). Ей передаются два параметра, дата документа и наименование выбранного товара. Эти параметры не обходимы для обращения периодическому регистру Цены.

В функции ТоварыНоменклатураПриИзмененииНаСервере() создается новая структура, в которую заносится имя товара, ререданное из вызывающей процедуры в виде параметра. Ведется обращение к регистру сведений Цены и методом ПолучитьПоследнее выбирается последняя (актуальная) цена из регистра. Выбранная цена передается в точку вызова.

Выполняется умножение выбранной цены на количество, результат заносистся в колонку Сумма.

Происходит обращение к функции ПолучитьСтавкуТовара(), передается один параметр, наименование выбранного товара, по которому выбирается ставкаНДС и передается в точку вызова.

Передается управление процедуре ТоварыСтавкаНДМПриИзменении(), которая обращается к функции ПолучитьСтавку() и процедуре ТоварыСуммаНДСПриИзменении(), рисунок 5.

Рисунок 5 - Тексты процедур

Для создания процедуры ТоварыСуммаНДСПриИзменении() откройте свойства заголовка СуммаНДС таблицы Товары и создайте процедуру обработки события при изменении. Функция ПолучитьСтавку() выбирает из найденного товара ставку НДС и делит выбранное значение на 100, чтобы получить значение в виде десятичной дроби и возвращается в точку вызова. Процедура ТоварыСуммаНДСПриИзменении() Значение Сумма складывает со значением СтавкаНДС и результат записывает в колонку Всего.

Описание процедуры ОбработкаПроведения(). С помощью запроса к БД, выполненного с помощью конструктора запросов, формируется текст запроса, где указывается какие реквизиты будут выбираться из табличной части документа. Строка                               РасходнаяНакладнаяТовары.Ссылка = &Ссылка                                                                                          указывает на то, что будут выбираться строки из табличной части конкретного документа. В состав запроса включено требование, чтобы по реквизиту КОЛИЧЕСТВО производилось суммирование, а по реквизитам ТОВАР и ЦЕНА производилась группировка. Это приводит к тому, что записи выбираются из табличной части,  совпадающие строки группируются, по реквизиту КОЛИЧЕСТВО рассчитывается сумма и вся эта информация размещается в оперативной памяти. Поэтому некоторые строки кода процедуры, сформированной конструктором необходимо откорректировать. В частности значения реквизитов ТОВАР, ЦЕНА, КОЛИЧЕСТВО будут выбираться из не из табличной части документа, а из оперативной памяти.
