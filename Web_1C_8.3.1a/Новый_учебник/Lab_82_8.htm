Требуется создать механизм, позволяющий пользователю произвольным образом описывать материалы и вести  учет в разрезе сделанных описаний. Для реализации этой проблемы разработчик должен поставить пользователю определенный механизм, позволяющий задавать эти характеристики. Такой механизм можно создать с помощью объекта конфигурации "План видов характеристик".

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Связь по типу обеспечивает  соответствие типа значений, вводимых в  поле ресурса Значение, и типа значений характеристики, выбранной в поле ВидСвойства.

 

 

 

 

Задавая тип ресурса как Характеристика.<Имя>  можно определить составной тип данных как он задан в плане видов характеристик СвойстваНоменклатуры.  Это говорит о том, ресурс регистра может принимать  значения различных типов, см. рис.7

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

 

Для обеспечения учета материалов по значениям характеристик необходимо изменить структуру регистра накопления ОстаткиМатериалов и документов ПриходнаяНакладная и ОказаниеУслугНовый

 

 

 

 

 

 

 

 

 

Указание свойства "Связи параметров выбора"  предназначено для того, чтобы после выбора номенклатуры в документе можно было выбрать только те свойства, которые заданы именно для этой номенклатуры.
