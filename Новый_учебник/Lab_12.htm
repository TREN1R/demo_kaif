Лабораторная работа №12

Работа с файлами *.DBF   
 

 

 

 

 

Работа с DBF в 1С 8.3 и 8.2 может быть полезна во многих случаях. Типичные примеры — обмен данными с другими учетными системами, загрузка классификаторов и так далее. DBF — формат хранения файлов баз данных, очень распространенный в России и мире.

Система 1С без проблем умеет работать с такими файлами с помощью встроенного объекта системы XBase.

 

  

     

Техническое задание

  1. Создать файл формата DBF
  2. Создать справочник с реквизитами для копирования в него файла DBF
  3. Создать обработку для чтения файла DBF и переноса информации из этого файла в справочник

.  Часть 1.

Рекомендации к решению

1.   Откройте MS Access и создайте в его среде таблицу с двумя колонками FIO и ADRESS, рисунок 1 и сохраните эту таблицу с именем FILE.

Рисунок 1 - Таблица с данными

  2. Экспортируйте созданную таблицу в виде файла DBasе, рисунок 2. Файл сохраните в базе 1С.

  Рисунок 2 - Создание файла DBF

   3. Создайте в 1С справочник "Сотрудники проба"с реквизитами для занесения в них информации из DBF файла. Так как DBF файл содержит таблицу, состоящую из двух колонок, то следует создать один реквизит "Адрес" строка с неограниченной длиной, а для поля "Наименование" установите длину 40 для внесения в него имени и фамилии.

   4. Создайте в 1С обработку  "ПроверкаDBF" без реквизитов с  единственной кнопкой "Выполнить", кнопку свяжите с процедурами, рисунок 3. Кнопке присвойте заголовок "Прочитать DBF".

 

Рисунок 3 - Процедуры, связанные с кнопкой "Выполнить"

   Описание процедур. 1)  Процедура ПутьКФайлуНачалоВыбора(). Процедура формирует диалоговое окно для открытия файла. Открытие файла задается переменной РЕЖИМ. Для диалогового окна можно устанавливать фильтры, которые определяют особенности работы с диалогом: фильтр "Фильтр" задает расширение файла, который может быть открыт, фильтр "МножественныйВыбор" определяет можно ли за одно открытие диалога выбрать несколько файлов, фильтр "ПредварительныйПросмотр" устанавливает режим вывода в окне диалога содержимого открываемого файла, фильтр "ИндексФильтра" определяет индекс активного фильтра, фильтр "ПроверятьСуществованиеФайла"  сдержит признак проверки существования выбираемого файла.

                                       2)  Процедура СоздатьСтрокиСправочника(). Для создания строк справочника из файла DBF требуется создание объекта XBase. Затем для этого файла устанавливается используемая кодировка OEM. В 1С можно использовать кодировку OEM - кодировка Dos и кодировка ANSII - кодировка Windows. Для чтения Кириллицы нужна кодировка OEM. Конструкция Xbase.Первая() позиционирует считывающее устройство на первой записи файла. Конструкция Xbase.Следующая() позиционирует считывающее устройство на следующую запись файла; Конструкция Xbase.ВКонце() указывает на достижение метки конца файла. Конструкция Справочник.СоздатьЭлемент() формирует новую строку справочника.

   5.  Зайдите в режим пользователя, выполните обработку, а затем откройте справочник "Сотрудники проба", он будет заполнен информацией, считанной их файла DBF, рисунок 4.

 

Рисунок 4 - Заполненный справочник

   6. Откройте MS Access и импортируйте в него файл Excel "ОКАТО.xls". В результате получится таблица сформированная из файла Excel, рисунок 5.  Полученную таблицу сохраните в формате DBF. (ОКАТО - общегосударственный классификатор административно-территориальных объектов, файл находится в корне учебника Web_1C_8.3).

Рисунок 5 - Импорт файла Excel в таблицу MS Access

   Самостоятельно. Создайте в 1С справочник "Классификатор ОКАТО" с реквизитами совпадающими с колонками сформированного файла (рисунок 5) и перенесите информацию из DBF файла в созданный справочник. Прежде чем осуществлять перенос наименования колонок напишите на латинице.

         Примечание. В таблицах DBF наименования колонок необходимо писать на латинице.

   Рекомендации к решению

   7.  Включите в состав обработки ПроверкаDBF дополнительную кнопку для создания и записи файла DBF. Файл будет заполняться информацией из справочника "Контрагенты". Как было сказано выше наименования колонок создаваемого файла должны быть написаны на латинице. Следовательно определим имена колонок создаваемого файла, таблица 1.

Описание процедур. 1)  Процедура Создать() аналогична процедуре ПутьКФайлуНачалоВыбора() с той разницей, что режим диалога будет СОХРАНЕНИЕ.

                                       2)  Процедура СоздатьНаСервере(). Для работы с файлом DBF требуется создание объекта XBase. Затем для этого файла устанавливается используемая кодировка OEM. В 1С можно использовать кодировку OEM - кодировка Dos и кодировка ANSII - кодировка Windows. Для чтения Кириллицы нужна кодировка OEM. Методом Добавить добавляются новые колонки в создаваемый DBF файл (имена колонок должны быть написаны на латинице). Затем из справочника Контрагенты выбираются последовательно все записи кроме групповых и данными  из них происходит заполнение колонок информацией. После заполнения очередной строки, она записывается. В конце работы методом ЗакрытьФайл() происходит формирование конечной метки файла и он закрывается.

  8. Войдите в режим пользователя, откройте обработку и с помощью кнопки Создать DBF сформируйте файл DBF с именем KTRA. Рекомендуется все создаваемые файлы сохранять в отдельной папке в рабочей базе данных.

   9.  Для проверки выполненной работы откройте файл KTRA.DBF, где можно увидеть таблицу с содержимым справочника "Номенклатура", рисунок 8

Рисунок 8 - Сформированный файл средствами 1С 8.3
