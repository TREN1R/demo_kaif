      Техническое задание.   Выполнить подготовительные работы для создания информационной системы:

  •  Создать пустую конфигурацию под именем "ООО Услуга"
  •  Задать роль, права и интерфейс
  • Создать Обработку "Загрузка".

Алгоритм обработки. Обработка "Загрузка" должна открываться во время загрузки режима пользователя. В составе обработки имеется поля для ввода даты. Если введенная дата и дата, взятая из системы, совпадают, то в этом случае окно обработки закрывается и пользователь может выполнять текущую работу. Если даты не совпадают, то в зависимости от введенного значения режим пользователя или будет закрыи или будет предложено ввести дату повторно.

   Описание процедуры №1

    1)  Методом ЭтотОбъект происходит обращение к реквизиту формы ДатаНачалаРаботы и в переменную ДатаПроба копируется информация из этого реквизита   ДатаПроба=ЭтотОбъект.ДатаНачалаРаботы;

  2)  Конструкцией Если   Тогда   Иначе   КонецЕсли производится ветвление алгоритма. В данном случае производится сравнение системной даты

             (   ТекущаяДата()  ) и даты, введенной пользователем в поле обработки. Так как платформа поставляет дату в виде даты и времени в формате                           ДД.ММ.ГГГГ ЧЧ:ММ:СС , то производится фильтрация дат путем выборки первых десяти символов с помощью системной функции ЛЕВ, так как при сравнении время может не совпадать.        Лев(ДатаПроба,10)<>Лев(ТекущаяДата(),10)

В функцию ЛЕВ вводят два параметра, первый параметр содержит информацию, которую требуется отфильтровать, а второй указывает количество символов подлежащих выборке слева. Аналогично работает и функция ПРАВ.

   3)   Метод СООБЩИТЬ позволяет вывести в окно сообщений информацию, записанную в виде строки. Метод ВвестиЗначение  позволяет вывести на экран диалоговое окно для ввода информации, которую может проанализировать программа. Информация вводится в переменную БулеваПерем.  Первоначально этой переменной  присваивается значение НЕОПРЕДЕЛЕНО.  Затем, в методе ВестиЗначение указываем какой тип должен поступить  в эту переменную. Если в эту переменную введено значение ЛОЖЬ, то тогда работы пользователя будет прекращена. Если системная дата и дата, введенная пользователем совпадают, то в этом случае будет закрыто только окно обработки, а работа пользователя не прекратится, рисунок 10.

  Для работы функции ВестиЗначение необходимо открыть свойства Конфигурации и в разделе СОВМЕСТИМОСТЬ разрешить использование модальности, рисунок 11. Если окно открыто в модальном режиме, то в этом случае доступ ко всем остальным формами элементам будет закрыт. Проверьте это утверждение на практике. Когда в режиме пользователя введена не верная дата, открывается окно для ввода информации ДА или НЕТ. Это окно единственное активное на всем рабочем столе, то есть оно открыто в модальном режиме.

РОЛЬ - это специальный объект ветки ОБЩИЕ, предназначенный для описания различных полномочий ПОЛЬЗОВАТЕЛЯ, которые заключаются в доступе к информации, обрабатываемой системой. Обычно роли указываются для каждого вида деятельности. Для каждого Пользователя может быть назначена одна или несколько ролей. если у Пользователя несколько ролей, то в этом случае ему предоставляется доступ к информации по следующему сценарию:

- если хотя бы в одной роли есть разрешение, то доступ будет открыт;

- если во всех ролях имеется запрещение, то доступ будет закрыт.

 

 

 

 

 

Директивы компиляции

Каждая процедура и функция модуля формы, модуля команды и общего модуля управляемого приложения предваряется директивой компиляции, определяющей среду исполнения данной процедуры. Директива предваряется символом "&". Допустимые директивы:

  • &НаКлиенте  — определяет клиентскую процедуру (функцию);
  • &НаСервере  — определяет серверную процедуру (функцию);
  • &НаСервереБезКонтекста - определяет серверную процедуру (функцию), исполняемую на сервере вне контекста формы
  • &НаКлиентеНаСервереБезКонтекста - определяет процедуру (функцию), исполняемую в модуле формы на клиенте и на сервере, не имеющую доступа к контексту формы, данным формы, переменным, но имеющую доступ к процедурам и функциям общих модулей
  • &НаКлиентеНаСервере — определяет процедуру (функцию), исполняемую в модуле команды, выполняемую на клиенте и на сервере
