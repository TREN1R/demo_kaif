Инструменты отладки прикладного решения

 
Практическое задание

Контрольные вопросы

            Отладчик является встроенным в конфигуратор инструментом и является вторым по значимости инструментом разработчика, после текстового редактора, облегчающий разработку и отладку программных модулей.

            Чтобы иметь возможность отлаживать код на встроенном языке, нужно обеспечить запуск приложения, в котором исполняется код, в отладочном режиме. 

            В общем, можно сказать что отладчик – это инструмент, который позволяет увидеть внутренние процессы и управлять работой системы не на интерфейсном уровне (как у пользователей), а на уровне алгоритмов, т.е. представляет своего рода электронный микроскоп  для прикладного решения c набором микроинструментов.

В платформе "1С:Предприятие 8.3" существует возможность отладки прикладного решения в различных режимах работы приложения, рисунок 1.

Рисунок 1 – Настройка режимов отладки

Передача управления отладчику это механизм, который позволяет разработчику остановить приложение  во время исполнения алгоритма в прикладном решении для решения разных задач. Например, для написания новых строк алгоритма с помощью отладчика, или для проверки работоспособности алгоритма, а может для исправления ошибок. Для каждого случая существует свой способ остановки:

-            с помощью точек останова;

-            по ошибке;

-            на первой строке исполняемого кода.

Использование точек останова

            В любом модуле можно установить специальный маркер - точку останова, - при достижении которой исполнение программного модуля останавливается (без выполнения строки, на которой стоит маркер) и управление передается отладчику. Точку останова можно установить в любой строке модуля, в любой момент работы с Отладчиком. В случае, если строка, на которой устанавливается точка останова, не  содержит операторов (например, пустая строка), содержит неисполняемый текст (например, заголовок процедуры или функции, определение переменных) или является продолжением оператора, начатого на предыдущих строках, положение точки останова будут автоматически скорректировано.

            Для управления точками останова используются команды меню "Отладка" или команды из контекстного меню, вызываемого из строки.

В левой колонке текста модуля специальными значками показываются текущая точка останова, места останова и состояние точек останова, рисунок 2.

Рисунок 2 – Маркировка точек останова

            Точки останова могут быть безусловными или с условием. При достижении безусловной точки останова исполнение программного модуля останавливается в любом случае. Если подвести указатель мыши к переменной, которая к этому моменту была выполнена, то всплывет табло с содержимым этой переменной. Для пошагового продвижения можно воспользоваться кнопкой F11, рисунок 3.

Рисунок 3 – Маркеры выполняемой отладки

 

При достижении точки останова с условием, выполнение программного модуля останавливается только в том случае, если заданное условие истинно, рисунок 4-5.

Остановка по ошибке

В платформе предусмотрена возможность использовать режим Остановка по ошибке. Команда "Отладка — Остановка по ошибке…" устанавливает или снимает установку остановки в случае, если в процессе выполнения возникает ошибка. При этом на экран выводится предупреждение "Ошибка времени выполнения:" и приводится поясняющий текст о месте и причине возникновения ошибки.

Если установлен флажок "Останавливаться только на ошибках", содержащих текст, то отладка будет остановлена в месте ошибки (если сообщение об ошибке содержит указанную в табличном поле подстроку). Если этот флажок не установлен, то остановка по ошибке будет происходить независимо от текста сообщения об ошибке. При этом указанная подстрока будет учитываться, только если флажок напротив нее установлен.

На месте возникновения ошибки устанавливается текущая точка исполнения текущего предмета отладки, а предмет отладки, в котором произошла ошибка, становится текущим. В этом режиме отладка может быть остановлена либо при каждой возникающей ошибке, либо при ошибке, в тексте которой присутствует подстрока, указанная разработчиком. Подстроки интересующих ошибок указываются в настройках отладки.

Пошаговое выполнение

            После того, как при достижении точки останова управление прикладным решением передано отладчику, существует возможность дальнейшего исполнения модуля в нескольких режимах:

-            продолжение свободного выполнения модуля (F5);

-            пошаговое выполнения (F11);

-            исполнение вызова функции или процедуры (F10);

-            прерывание пошагового исполнения функции или процедуры (Shift+F11);

-            выполнения модуля до той строки, на которой стоит курсор (Shift+F10).

 

Просмотр значений

            Во время пошагового выполнения существует возможность просматривать значения переменных, содержащихся в модуле, и рассчитывать произвольные значения. С помощью табло и диалога "Выражение" вы можете получить значения интересующих выражений.
