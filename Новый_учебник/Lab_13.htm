   Описание процедуры ЗаписьXML. Создается диалог выбора файла с режимом СОХРАНЕНИЕ.  Ссылка на созданный диалог размещается в переменной ДИАЛОГ. Для диалога устанавливается два свойства: Фильтр - указывает какого типа файл будет выбираться и МножественныйВыбор устанавливается в значение ЛОЖЬ, говорит о том, что выбираться будет только один файл. Из диалога выбирается полное имя файла и управление передается процедуре ЗаписьXMLНаСервере.

  Описание процедуры  ЗаписьXMLНаСервере.  Создается новый элемент ЗаписьXML, ссылка на него заносится в переменную Запись. Производится открытие доступа к выбранному файлу.  Методом ЗаписатьОбъявлениеXML()  в начале документа формируется запись <?xml version="1.0" encoding="UTF-8"?>, свидетельствующая об объявлении документа. Запись объявления XML должна выполняться перед другими операциями записи XML. Строка Запись.ЗаписатьНачалоЭлемента("Контрагенты")  формирует начало элемента.  Строка  Справочники.Контрагенты.Выбрать Формирует выборку элементов справочника по заданным условиям, выборка строк справочника организуется в цикле. Так как справочник иерархический, строки с заголовками группы не учитываются. Затем формируется атрибут "Код", а все остальные реквизиты пишутся в виде сплошного текста. Информация реквизита БАНК перед записью преобразуется в строку. Методом ЗаписатьКонецЭлемента() формируется конец тэга. А после окончания цикла формируется конечный элемент файла аналогичной строкой.

  Описание процедуры ПрочестьXML. Эта процедура аналогична процедуре ЗаписьXML с той разницей, что при создании диалога выбора файла режимом диалога выбора файла будет ОТКРЫТИЕ

  Описание процедуры ПрочестьXMLНаСервере(). Особенностью этой процедуры является необходимость разделения текстовой информации на отдельные реквизиты. Проблемой является то, что вся информация записана сплошным текстом, а ее следует разделить на реквизиты. В связи с тем, что реквизит "Наименование" не может иметь фиксированную длину, потребуется наименование выбирать по одному символу и каждый символ проверять на его код. В результате проверки с помощью метода КодСимвола программе возвращается Код  в соответствии с кодировкой Unicode в десятичном формате. Анализ текста, входящего в состав наименования, показал, что в него помимо букв Кириллицы могут входить специальные символы, в нашем случае это кавычки ", дефис -, пробел. Анализ кодировки Unicode показал, что цифры кодируются в диапазоне с

Кодировка цифр: 0 - 48; 1 - 49;…9 – 57. Пробел – 32; кавычки “ – 34; дефис – 45. Служебные символы от 32 до 47 и от 58 до 64. С кода 65 пошла латиница. Кириллица занимает диапазон с 1040 по 1103.

 Юникод (Unicode), это многоязычный, основанный на ASCII стандарт кодирования символов, а также, связанное с ним, семейство многобайтных кодировок. 

Полную таблицу UNICODE можно посмотреть по ссылке  http://foxtools.ru/Unicode

Так как в нашем случае в XML файле имеется единственный реквизит "Код", который нам не нужен (при формировании строки справочника система сама присвоит номер создаваемым строкам), то мы его обходим и определяем тип узла. Если тип узла текст, тогда производится считывание текста по одному символу и проверка его на цифру и на служебные знаки. Метод ПРЕРВАТЬ позволяет выйти из цикла как только заданное условие не выполняется. Строки предназначены для заполнения реквизитов справочника. Функция СРЕД позволяет выбрать из заданной области нужное количество символов начиная с номера символа, заданного переменной число. Если количество символов для выборки не задано, то выбираются все символы до конца строки, что видно при заполнении реквизита БАНК. Последняя строка из приведенного фрагмента записывает в справочник сформированную строку.
